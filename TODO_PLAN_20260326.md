# Post-E2E Improvements: Bug Fix, Regression Tests, Documentation

## Context

During implementation of `tests/e2e/test_demo_processing.py`, three categories of issues were discovered:

1. **MissingGreenlet bug** in `POST /api/series` — accessing `result.study.patient_id` after `session.refresh()` expires the relationship
2. **No test coverage** for `input_files`/`output_files` computed fields after DB round-trip — the bug where raw dicts from JSON column caused `AttributeError: 'dict' object has no attribute 'role'` was already fixed in `record.py`, but nothing prevents regression
3. **Documentation gaps** — CLAUDE.md files don't warn about `refresh()` expiring relationships or JSON columns returning raw dicts

## 1. Fix MissingGreenlet in `study_service.create_series()`

**File:** `src/services/study_service.py:288-290`

**Root cause:** `create_series()` fetches `study` at line 282, assigns `series.study = study` at line 289, then calls `self.series_repo.create(series)` at line 290. The base `create()` does `session.refresh(entity)` which expires all relationships. When the router at `src/api/routers/study.py:185` accesses `result.study.patient_id`, it triggers a sync lazy load → MissingGreenlet.

**Fix:** Re-attach the already-fetched `study` object after create:

```python
# Current (line 290):
return await self.series_repo.create(series)

# Fixed:
result = await self.series_repo.create(series)
result.study = study  # Re-attach: refresh() expired the relationship
return result
```

Zero-risk, zero extra queries. The `study` variable is still valid in scope.

## 3. Documentation Updates

### 3A. `src/models/CLAUDE.md` — JSON column warning

Add after the "JSON Columns" section (after line ~85):

> **Warning:** JSON columns return raw Python dicts/lists, not Pydantic models. After loading from DB, `RecordType.file_registry` contains `list[dict]`, not `list[FileDefinition]`. Code iterating `file_registry` directly must handle both types via `FileDefinition(**item) if isinstance(item, dict) else item`. The `input_files`/`output_files` computed properties handle this automatically.

### 3B. `src/repositories/CLAUDE.md` — refresh behavior

Add new section after "Base Repository API" (after line ~22):

> **Refresh and Relationship Expiration**: `create()` and `update()` call `session.refresh(entity)` after commit, which **expires all loaded relationships**. Accessing a relationship attribute after these methods will trigger a sync lazy load → `MissingGreenlet` in async contexts.
> Safe patterns: (1) re-assign the relationship object, (2) use only scalar FK fields, (3) re-query with eager loading.
