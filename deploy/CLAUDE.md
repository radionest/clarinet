# Deploy Infrastructure

## Overview

Shell-based deploy pipeline for KVM/libvirt VMs. No Ansible/Docker dependency — scripts are directly reusable on any Ubuntu 24.04 server.

## Structure

```
deploy/
  vm/          VM lifecycle (vm.sh, vm.conf, cloud-init)
  install/     Production installer scripts (run on target)
  systemd/     Service unit files
  nginx/       Reverse proxy config + SSL
  test/        Smoke tests + E2E orchestrator
```

## Key Conventions

- **All scripts idempotent** — safe to re-run
- **vm.conf** is the single source of truth for VM parameters and `PATH_PREFIX`
- **Placeholders**: `__SSH_PUBLIC_KEY__` in cloud-init, `__PATH_PREFIX__` in nginx conf — replaced at deploy time
- **Sub-path deployment**: set `PATH_PREFIX="/nir_liver/"` in vm.conf; nginx strips prefix via `proxy_pass` trailing slash, `root_url` tells FastAPI about the prefix
- **Secrets**: auto-generated by `generate-settings.sh` via `openssl rand`, never committed

## Commands

```bash
make vm-create    # Boot VM from cloud image
make vm-deploy    # Build wheel + deploy to VM
make vm-smoke     # Run smoke tests
make vm-test      # Full E2E cycle
make vm-destroy   # Tear down VM
make vm-reimage   # Destroy + recreate
```

## Host Dependencies

`sudo apt install libvirt-daemon-system virtinst cloud-image-utils qemu-utils`
