# Исследование диагностической эффективности методов лучевой диагностики в выявлении метастазов печени

## Цель исследования

Оценка диагностической эффективности различных методов лучевой диагностики в выявлении метастазов печени.

## Участники исследования

1. **Главный исследователь (монитор)** — 1 рентгенолог с опытом работы в лучевой диагностике печени не менее 5 лет
2. **Группа А** — врачи-рентгенологи с опытом работы в лучевой диагностике печени не менее 5 лет
3. **Группа Б** — 2 врача-рентгенолога с опытом просмотра КТ ангиографии не менее 1 года
4. **Группа В** — рентгенхирурги с опытом работы в печени не менее 5 лет
5. **Эксперт** — специалист, создающий и обновляющий эталонную мастер-модель

## Изучаемые модальности

Пациент при соответствии критериям включения поступает на операцию. За 3 дня до операции ему выполняются:
- **КТ** — компьютерная томография с внутривенным контрастированием
- **МРТ** — магнитно-резонансная томография с внутривенным контрастированием
- **КТ-АГ** — КТ ангиография
- **МРТ-АГ** — МРТ ангиография
- **ПДКТ-АГ** — позитронно-эмиссионная компьютерная томография ангиография

Дополнительно:
- **КТ-архив** — предыдущие КТ, выполненные до химиотерапии (может быть несколько исследований)

Каждая модальность представлена **одним** исследованием (кроме КТ-архива). Сегментация непосредственно на КТ-архив не выполняется, но эти данные доступны врачу при сегментации КТ с расширенным контекстом.

## Основная концепция: мастер-модель

**Мастер-модель** — эталонная сегментация всех выявленных очагов у пациента, где каждый очаг расположен в отдельной зоне интереса (ROI) с уникальным номером.

Основой мастер-модели является первая КТ пациента с завершённой сегментацией. Координатная сетка (число пикселей, размеры пикселей, точка отсчёта) определяется эталонной серией из первичной оценки качества и остаётся неизменной.

Мастер-модель создаётся один раз и дополняется новыми очагами по мере их обнаружения на исследованиях других модальностей. Каждый новый очаг получает уникальный номер.

Для каждой модальности (МРТ, КТ-АГ, МРТ-АГ, ПДКТ-АГ) создаётся **проекция** мастер-модели — перенос очагов из координатного пространства КТ в координатное пространство этой модальности.

## Этапы исследования

### 1. Оценка качества (ручное)

**Кто выполняет:** врачи-рентгенологи (2 независимые оценки)

**Когда:** автоматически выдаётся задача при поступлении исследования в базу

**Что определяется:**
- Пригодность исследования для дальнейшей работы
- Тип модальности (КТ, МРТ, КТ-АГ, МРТ-АГ, ПДКТ-АГ, КТ-архив)
- **Эталонная серия** — серия с наименьшим числом артефактов и наибольшим числом анатомических ориентиров, на которой будет выполняться разметка очагов

Поиск очагов осуществляется на всех доступных сериях, но разметка выполняется только на эталонной серии.

**Результат:** исследование либо допускается к следующим этапам, либо отклоняется

### 2. Анонимизация (автоматически)

Выполняется автоматически при прохождении проверки качества. Все персональные данные пациента заменяются на анонимные идентификаторы.

### 3. Сегментация очагов (ручное)

**Кто выполняет:** врачи соответствующей специализации в зависимости от модальности
- КТ → врачи со специализацией в КТ
- МРТ → врачи со специализацией в МРТ
- КТ-АГ → врачи со специализацией в ангиографии
- МРТ-АГ → врачи со специализацией в МРТ
- ПДКТ-АГ → врачи со специализацией в ПДКТ

**Когда:** после прохождения оценки качества и анонимизации

**Количество участников:** каждое исследование размечается 1-4 врачами независимо

**Доступная информация:**
- Для КТ (стандартный контекст): только текущее исследование
- Для КТ (расширенный контекст): текущее исследование + все КТ-архивы пациента
- Для других модальностей: только текущее исследование

**Процесс:**
1. Врач просматривает анонимизированное исследование в OHIF viewer
2. Загружает эталонную серию в 3D Slicer
3. Размечает очаги в трёх категориях:
   - **mts** (metastasis) — метастазы
   - **unclear** — неопределённые образования
   - **benign** — доброкачественные образования

**Результат:** файл сегментации в формате NRRD

**Особенности:**
- Сегментации разных врачей независимы и не блокируют друг друга
- Каждый врач создаёт отдельный файл сегментации
- Сегментация выполняется на эталонной серии, но врач может просматривать все доступные серии

### 4. Построение мастер-модели (автоматически)

**Когда:** после завершения первой КТ-сегментации (любого из врачей)

**Процесс:**
- Создаётся один раз на основе первой завершённой КТ-сегментации
- Координатная сетка определяется эталонной серией из оценки качества
- Каждый очаг из сегментации размещается в отдельной ROI с уникальным номером (1, 2, 3, ...)
- Первая проекция (на ту же серию КТ) является копией мастер-модели

**Результат:** файл master_model — базовая эталонная сегментация

### 5. Построение проекции мастер-модели (ручное)

**Кто выполняет:** эксперт

**Когда:** после создания мастер-модели для каждой новой модальности (МРТ, КТ-АГ, МРТ-АГ, ПДКТ-АГ)

**Процесс:**
1. Эксперт получает задачу с двумя слоями в 3D Slicer:
   - **master_model** — изображение мастер-модели с ROI, которые нужно перенести
   - **projection** — пустые ROI с соответствующими номерами на целевой модальности
2. Эксперт вручную переносит каждый очаг в координатное пространство новой модальности
3. Если очаг не виден на целевой модальности — ROI остаётся пустой

**Результат:** файл projection — проекция мастер-модели на координатное пространство данной серии

**Особенности:**
- Одна проекция на серию
- Если мастер-модель ещё не создана, задача блокируется до её появления

### 6. Сравнение проекции и сегментации (автоматически)

**Когда:** после завершения каждой сегментации и наличия проекции для данной серии

**Процесс:**
- Автоматическое попиксельное сравнение файлов projection и segmentation
- Определение пересечений между ROI
- Если есть пересечение — считается, что это один и тот же очаг
- Если нет пересечения — очаг пропущен в одной из разметок

**Результат:**
- Список **false negative** — очаги из проекции, пропущенные врачом при сегментации (очаги с номерами из мастер-модели)
- Список **false positive** — очаги, размеченные врачом, но отсутствующие в проекции мастер-модели

**Действия по результатам:**
- Если есть false positive → автоматически создаётся задача "Обновление мастер-модели"
- Если есть false negative → создаётся задача "Пересмотр" для врача

**Особенности:**
- Выполняется для каждого врача независимо (если 4 врача — 4 сравнения)
- Результаты видны всем участникам для анализа

### 7. Пересмотр (second_review) (ручное)

**Кто выполняет:** врач, чья сегментация расходится с проекцией мастер-модели

**Когда:** при обнаружении расхождений (false negative или false positive) в сравнении

**Процесс:**
1. Врач получает задачу с двумя слоями в 3D Slicer:
   - **master_projection** — очаги из проекции, которые врач пропустил (каждый под своим номером)
   - **output** — 4 пустые категории: metastasis, unclear, benign, invisible
2. Врач классифицирует каждый пропущенный очаг, переназначая его из первого слоя во второй:
   - **metastasis** — подтверждает метастаз
   - **unclear** — неопределённое образование
   - **benign** — доброкачественное образование
   - **invisible** — очаг не виден на данной модальности

**Результат:** файл second_review с классификацией расхождений

**Особенности:**
- Создаётся для конкретного врача
- При повторном обнаружении новых расхождений (после обновления мастер-модели) задача инвалидируется и создаётся заново
- Предыдущие результаты пересмотра сохраняются и объединяются с новыми

### 8. Обновление мастер-модели (ручное)

**Кто выполняет:** эксперт

**Когда:** при обнаружении false positive (очаги, размеченные врачом, но отсутствующие в мастер-модели)

**Процесс:**
1. Эксперт получает задачу с:
   - Мастер-моделью и КТ, на основе которой она создана
   - Сегментацией с нового исследования и соответствующим изображением
   - Новые очаги выделены отдельным цветом
2. Эксперт оценивает новые очаги и при необходимости добавляет их на мастер-модель
3. Новым очагам присваиваются следующие свободные уникальные номера

**Результат:** обновлённый файл master_model

**Последствия обновления:**
- Все существующие проекции инвалидируются (так как изменился эталон)
- Создаются новые задачи для экспертов на повторное построение проекций
- При повторном сравнении могут быть выявлены новые расхождения для врачей

**Особенности:**
- Максимум одна задача обновления на пациента одновременно
- Координатная сетка мастер-модели не меняется
- Обновление не удаляет существующие очаги, только добавляет новые

## Множественные участники

Каждое исследование может быть размечено несколькими врачами (от 1 до 4):
- При создании задачи автоматически создаётся минимальное количество записей (обычно 1)
- Дополнительные врачи могут быть назначены вручную или автоматически
- Каждый врач работает независимо:
  - Имеет свой файл сегментации
  - Имеет свой жизненный цикл задачи
  - Имеет свои результаты сравнения
  - Имеет свою задачу пересмотра при расхождениях
- Сегментации разных врачей не блокируют друг друга

## Блокировка и порядок выполнения

**Параллельное выполнение:**
- Сегментации всех модальностей выполняются параллельно
- Сегментации разных врачей не зависят друг от друга
- Исследования могут поступить пакетом — порядок завершения не гарантирован

**Блокировка задач:**
- Построение проекции блокируется до создания мастер-модели
- Сравнение блокируется до завершения сегментации и создания проекции
- Пересмотр создаётся только при наличии расхождений

**Создание мастер-модели:**
- Создаётся по первой завершённой КТ-сегментации (любого из врачей)
- Не зависит от завершения других модальностей

## Обработка race conditions

**Проблема:** мастер-модель может быть обновлена в процессе создания проекции

**Решение:**
- При создании проекции сохраняется контрольная сумма (хэш) используемой мастер-модели
- При завершении проекции хэш сравнивается с текущим файлом мастер-модели
- При несовпадении проекция инвалидируется и создаётся новая задача
- Потери работы не происходит — проверка только при завершении задачи

## Пример последовательности действий

1. **КТ:** первичная оценка качества → определение эталонной серии → сегментация врачом → создание мастер-модели → проекция на ту же серию (копия)
2. **МРТ:** первичная оценка качества → определение эталонной серии → сегментация врачом → создание проекции мастер-модели экспертом → автоматическое сравнение
3. **Расхождение:** обнаружен 1 очаг на сегментации МРТ, отсутствующий в проекции (false positive) → создаётся задача обновления мастер-модели
4. **Обновление:** эксперт добавляет новый очаг на мастер-модель → все существующие проекции инвалидируются
5. **Повторные проекции:** эксперты заново размечают проекции для всех модальностей
6. **Пересмотр:** для врачей, чьи сегментации расходились с обновлённой проекцией, создаются задачи пересмотра
7. **КТ-АГ:** первичная оценка → сегментация → проекция → сравнение → расхождений не выявлено
8. **ПДКТ-АГ:** аналогичный цикл → расхождений не выявлено
